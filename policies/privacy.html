Privacy Policy

Last updated: 18 June 2026.

Preferred privacy boundary

BOS is intended to use compact metadata rather than raw project material. Local project memory should remain in project-controlled files unless a future feature explicitly requests consent for a narrower transfer.

Data BOS may collect with consent

  • Account, app, installation, entitlement, and agreement acceptance records.
  • Compact runtime events such as command type, capability used, rule version, status, blocked reason, and error category.
  • Sanitized resolved-pattern metadata, such as platform, issue category, capability used, outcome, and reusable advisory tags.
  • Support contact details you provide, such as email address and app type.

Data BOS should not receive by default

  • Raw source code, secrets, API keys, credentials, payment data, user content, personal records, database exports, raw logs, raw evidence records, full prompts, or full private project memory.

Retention and deletion

Production BOS should define retention, deletion, export, and correction processes before paid public launch. Until those controls are active, public paid launch remains blocked by BOS governance.
