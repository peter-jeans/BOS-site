BOS connection model

How BOS gives AI builders memory without taking over the app.

BOS sits beside Base44, GitHub, Codex, and other AI coding tools. It supplies current governance, app memory, scans, prompts, and receipts while keeping private source code and secrets out of Cloud BOS by default.

The two beta workflows

Direct app builders use Cloud BOS as a lightweight advisory layer. Code workflows add GitHub and localBOS/Codex so source changes, verification, and receipts can be handled in the repo.

Base44 direct workflow

Base44 app AICONTROL, app spec, project receipts, local governance pointers, surfaced build/runtime issues.
<-->
Cloud BOS Current rules, onboarding packs, scans, platform advisories, capability checks, upgrade metadata.

Code and GitHub workflow

Base44 or AI builder Builds, prompts, app changes, surfaced errors.
<-->
GitHub Source of truth, branches, commits, deployment evidence.
<-->
Codex / localBOS Reads repo, applies patches, runs tests, writes receipts, protects local project truth.
<-->
Cloud BOS Canonical rules, updates, advisories, capability metadata, compact resolved-pattern intake.

What lives where

Local project memory

Each app keeps its own project truth: app spec, decisions, receipts, local constraints, prior fixes, user preferences, and exceptions. This is the working case law for that app.

Cloud BOS guidance source

Cloud BOS publishes current rules, onboarding packs, capability lists, scan guidance, platform lessons, and upgrade metadata. It should receive compact metadata, not private project material by default.

AI builder execution

Base44, Codex, and other AI tools still do the building. BOS improves the prompt, route, evidence standard, and follow-up scan so the AI has a narrower and better-governed job.

Receipts and learning

When a problem is resolved, BOS can record what happened and, with onboarding consent, share a privacy-safe resolved pattern back to Cloud BOS for future advisories.

Privacy boundary

The beta design is intentionally conservative: Cloud BOS should help apps use better rules without needing to see the whole app.

Should stay local by default: source code, secrets, credentials, user content, raw logs, raw receipts, full prompts, full local project memory, and private development-pack material.
May be shared with consent: compact capability usage, resolved issue type, platform/runtime category, advisory outcome, version metadata, and non-sensitive receipt summaries.

Why this helps

Less rediscovery

The AI should not need to re-learn the app every chat. BOS keeps the app contract and prior decisions visible.

Better routes

Not every issue should become a broad build. BOS can steer toward scan, fix, verify, block, refresh, upgrade, or ask.

Reusable patterns

Hard-won fixes such as mobile viewport handling, backend persistence, Android GPS cold starts, and iOS sensor constraints become advisories for future apps.
